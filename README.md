# 2026_TRIPLES_TEAM_4_FE
대학 연합 프로젝트 TripleS FE 4팀 레파지토리입니다.
 
---
 
## 📝 개발 규칙(Convention)
 
 
### 🛠 태그 (커밋&PR 공통)
- **feat**: 신규 기능 추가
- **fix**: 버그 수정
- **chore**: 환경 설정, 라이브러리 추가, 폴더 구조 변경 (이슈 번호 생략 가능)
- **docs**: 문서 수정 (README 등)
- **style**: 기능 변경 없는 시각적 스타일 개선 (오타 수정, 들여쓰기 등)
- **refactor**: 코드 리팩토링 (기능 변경 없는 구조 개선)
   
---
  
### ✍️ 작성 양식
- **커밋 메세지 / PR 제목** : 태그명: 기능 내용 (#이슈번호)
- **브랜치명** : 태그명/#이슈번호-기능명
  
---
  
### 🚀 브랜치 및 PR 전략
- **작업 단위별로 브랜치를 생성**하며 완료 후 **main으로 PR을 요청**합니다.
- **UI 변경**이 있는 경우 PR 본문에 **스크린샷을 첨부**합니다.
- 상세 양식은 **PR 템플릿을 참고**해주세요.
  
---
  
### 📁 폴더 구조 (Package Structure)
- **ui**: 화면(Activity, Fragment, Adapter 등)
- **data**: 데이터 통신 및 API 관련 (Retrofit 등)
- **model**: 데이터 규격(DTO 등)
- **utils**: 공통 도구 및 함수
- **base**: 기본 틀. 공통 기능 클래스
  
---
  
### 📦 주요 라이브러리
- **Retrofit2 & Gson** (백엔드 API 통신 및 데이터 변환)
- **Glide** (이미지 로딩)
- **Navigation Component** (화면 흐름 관리)
- **ViewModel & LiveData**(수명 주기 관리)
- **ViewBinding**(레이아웃 연결)
